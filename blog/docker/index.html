How Docker works

Docker uses linux kernels two building blocks/primitives when building containers

  1. Namespaces: about isolation, carve the os in multiple isolated virtual operating system,(bit like hypervisors and vms). We use namespaces to take a single os with all its resources like file systems, process tress, users .. and carve all of that up into multiple virtual oses called containers. Each container gets its own virtualized/containerized root file system, its own process tree(starting from pid 1/systemd), its own networking interface, its own root user .. Linux namespaces comprises of (Process ID(pid), Network(net)/(isolated_network_stack,own ports,own NICs, IPs, routing_tables..), Filesystem/mount(mnt)(its own isolated root file system), Inter-proc comms(ipc)(processes in a single container access the same shared memory), UTS(uts)/hostname_for_each_container, User(user)/map_accounts_inside_of_a_container_to different user on the host(ex: mapping the container's root user to a non-privileged user on the host)
  2. Control Groups(cgroups): grouping objects/processes and setting limits, ex: container A is going to get this amount of CPU,memory,disk_IO_acess,network_bandwidth and then for containerB this amount and so on.

docker installation

Ubuntu #official way

sudo apt update
sudo apt install docker.io

#shows the docker version only
docker --version 

#shows extra version info(containerd, runc ..)
docker version    

#check to see whether docker is running or not
systemctl status docker

    #if the docker.service is disbled
    sudo systemctl enable docker  #docker service will start automatically when server/pc starts every time

    #if  docker is inactive
    sudo systemctl start docker

# Add user to docker group so that sudo needn't be typed everytime
sudo usermod -aG docker [user_name/pawan]
chmod 660 /run/docker.sock
